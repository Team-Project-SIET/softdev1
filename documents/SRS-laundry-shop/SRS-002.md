## 1. บทนำ

### 1.1 วัตถุประสงค์ (Objectives)
เอกสารนี้จัดทำขึ้นเพื่อระบุข้อกำหนดความต้องการของ **ระบบบริหารจัดการร้านรับซักรีดเสื้อผ้า** โดยมุ่งเน้นแก้ปัญหาความล่าช้าในการทำงานและการสูญหายของข้อมูล โดยระบบมีวัตถุประสงค์หลักดังนี้:
1.  เพื่อพัฒนาระบบ Web Application สำหรับบริหารจัดการร้านซักรีดแบบครบวงจร
2.  เพื่อเพิ่มประสิทธิภาพในการรับ-ส่งคืนเสื้อผ้า และลดข้อผิดพลาดในการตรวจสอบจำนวน
3.  เพื่ออำนวยความสะดวกให้ลูกค้าสามารถตรวจสอบสถานะการซักได้ด้วยตนเอง
4.  เพื่อช่วยผู้ประกอบการในการคำนวณต้นทุน จัดการสต็อก และสรุปยอดรายรับ-รายจ่าย

### 1.2 ขอบเขตของโครงงาน (Scope)
ระบบจะครอบคลุมกระบวนการทำงานผ่านเว็บแอปพลิเคชัน โดยแบ่งขอบเขตดังนี้:
* **Front-end:** ส่วนติดต่อผู้ใช้งานสำหรับลูกค้า (ตรวจสอบสถานะ, ดูโปรโมชัน) และพนักงาน (จัดการออเดอร์)
* **Back-end:** ระบบจัดการฐานข้อมูล, การคำนวณราคา, และระบบสมาชิก
* **Modules:** ครอบคลุมตั้งแต่นำผ้าเข้าระบบ จนถึงการส่งคืนและชำระเงิน

## 2. ภาพรวมของระบบ

### 2.1 กลุ่มผู้ใช้งาน (User Characteristics)
ระบบแบ่งผู้ใช้งานออกเป็น 4 กลุ่มหลัก ดังนี้:
1.  **ผู้ดูแลระบบ (Admin/Owner):** มีสิทธิ์เข้าถึงทุกส่วน ดูรายงาน Dashboard และจัดการสิทธิ์พนักงาน
2.  **พนักงานหน้าร้าน (Receptionist):** รับผ้า, คีย์ข้อมูล, รับชำระเงิน, ส่งมอบผ้า
3.  **พนักงานปฏิบัติการ (Operation Staff):** อัปเดตสถานะการซัก/รีด, เบิกใช้น้ำยา
4.  **ลูกค้า (Customer):** ตรวจสอบสถานะงาน, ดูประวัติการใช้บริการ, สะสมแต้ม



### 2.2 สถาปัตยกรรมระบบ (Proposed Tech Stack)
* **Web Framework:** (เช่น Laravel หรือ Node.js/Express)
* **Frontend:** (เช่น HTML5/Bootstrap หรือ Vue.js/React)
* **Database:** MySQL หรือ PostgreSQL
* **Deployment:** Cloud Server / Local Server

## 3. ความต้องการด้านฟังก์ชันการทำงาน



### 3.1 การจัดการลูกค้าและสมาชิก (CRM)
#### 3.1.1 ข้อมูลลูกค้า
* ระบบสามารถบันทึกข้อมูลลูกค้า (ชื่อ, โทรศัพท์, Line ID, ที่อยู่จัดส่ง)
* ระบบสร้างรหัสลูกค้าอัตโนมัติ (Customer ID) เพื่อความสะดวกในการอ้างอิง
* รองรับการค้นหาลูกค้าจาก เบอร์โทรศัพท์ หรือ ชื่อ ได้ทันที

#### 3.1.2 ระบบสมาชิก (Membership)
* **ระดับสมาชิก:** แบ่งเป็นลูกค้าทั่วไป และ สมาชิก VIP
* **การสะสมแต้ม:** คำนวณแต้มจากยอดใช้จ่าย (เช่น 25 บาท = 1 แต้ม) เพื่อใช้แลกส่วนลด
* **สิทธิประโยชน์:** ระบบคำนวณส่วนลดอัตโนมัติเมื่อเลือกลูกค้าที่เป็น VIP

### 3.2 การรับเสื้อผ้า (Order Management)
#### 3.2.1 การสร้างใบรับงาน (Job Order)
* บันทึกรายการผ้า แยกตามประเภท (เสื้อ, กางเกง, ผ้านวม) และบริการ (ซักรีด, ซักแห้ง)
* **ฟีเจอร์สำคัญ:** สามารถอัปโหลดรูปภาพตำหนิผ้าก่อนซัก เข้าระบบเพื่อเป็นหลักฐาน
* ระบุวันนัดรับ (Due Date) และความเร่งด่วน (Normal, Express)

#### 3.2.2 การออกเอกสาร
* พิมพ์ใบรับผ้า (Ticket) พร้อม **QR Code** ให้ลูกค้าสแกนเพื่อติดตามสถานะ
* พิมพ์ Tag กันน้ำสำหรับติดที่เสื้อผ้าเพื่อป้องกันการสลับเจ้าของ

### 3.3 การจัดการกระบวนการ (Process Management)
#### 3.3.1 การติดตามงาน (Tracking)
* แสดงสถานะปัจจุบันของแต่ละออเดอร์: `รอซัก` -> `กำลังซัก` -> `กำลังรีด` -> `เสร็จสิ้น/รอรับ`
* พนักงานปฏิบัติการสามารถกดเปลี่ยนสถานะงานผ่านแท็บเล็ตหรือมือถือได้

#### 3.3.2 การจัดการสต็อกและอุปกรณ์
* ตัดสต็อกน้ำยาซักฟอก/น้ำยาปรับผ้านุ่ม ตามจำนวนออเดอร์หรือการเบิกใช้จริง
* แจ้งเตือนเมื่อสินค้าคงคลังต่ำกว่าเกณฑ์ (Re-order Point)

### 3.4 การเงินและการชำระเงิน (Billing)
* คำนวณค่าบริการรวมโดยอัตโนมัติ (ค่าผ้า + ค่าบริการด่วน - ส่วนลด)
* รองรับการชำระเงินหลายรูปแบบ: เงินสด, โอนเงิน (แนบสลิป), QR Payment
* ออกใบเสร็จรับเงิน (Receipt) แบบย่อและเต็มรูป

### 3.5 ระบบรายงาน (Dashboard & Reporting)
ระบบสามารถแสดงผลข้อมูลเชิงวิเคราะห์ได้ดังนี้:
* **Daily Report:** สรุปยอดรับผ้าและยอดเงินประจำวัน
* **Order Status:** กราฟแสดงจำนวนงานที่ค้างอยู่ในแต่ละขั้นตอน (ซัก/รีด/แพ็ค)
* **Performance:** รายงานประวัติการใช้บริการของลูกค้าและการทำงานของพนักงาน

## 4. ความต้องการที่ไม่ใช่ฟังก์ชันการทำงาน

### 4.1 ประสิทธิภาพ (Performance)
* ระบบรองรับการใช้งานพร้อมกัน (Concurrent Users) ได้อย่างน้อย 50-100 คน
* เวลาตอบสนอง (Response Time) ของหน้าเว็บไม่เกิน 3 วินาที (บนเครือข่ายปกติ)
* ระบบสร้าง QR Code สำหรับใบรับผ้าได้ภายใน 1 วินาที

### 4.2 ความปลอดภัย (Security)
* มีระบบ Login/Logout และระบบ Session Timeout เมื่อไม่มีการใช้งานเกิน 30 นาที
* รหัสผ่านของผู้ใช้งานต้องถูกเข้ารหัส (Encryption) ในฐานข้อมูล
* มีการกำหนดสิทธิ์ (Permission) ชัดเจนระหว่าง "พนักงานหน้าร้าน" และ "เจ้าของร้าน"

### 4.3 ความสามารถในการใช้งาน (Usability)
* User Interface (UI) ออกแบบให้รองรับ **Responsive Design** ใช้งานได้ดีทั้งบน PC และ Mobile Browser
* เมนูและข้อความในระบบเป็นภาษาไทยทั้งหมด
* มีระบบแจ้งเตือน (Alert) เมื่อกรอกข้อมูลไม่ครบถ้วน

### 4.4 ความน่าเชื่อถือ (Reliability)
* ระบบมีฟังก์ชันสำรองข้อมูล (Database Backup) อัตโนมัติทุกวัน
* กรณีเซิร์ฟเวอร์ขัดข้อง ข้อมูลสถานะล่าสุดต้องไม่สูญหาย

## 5. ข้อจำกัดของระบบ (System Constraints)
1.  **การเชื่อมต่อ:** จำเป็นต้องใช้อินเทอร์เน็ตในการเข้าถึงระบบ (Web-based Application)
2.  **Platform:** ระบบทำงานผ่าน Web Browser เท่านั้น (ไม่รวม Native Mobile App)
3.  **Storage:** จำกัดขนาดไฟล์รูปภาพที่อัปโหลดไม่เกิน 10MB ต่อรูป เพื่อประหยัดพื้นที่ Server
4.  **Hardware:** ไม่รองรับการเชื่อมต่อกับเครื่อง POS หรือลิ้นชักเก็บเงินโดยตรง (ใช้งานผ่านหน้าเว็บ)
5.  **Regulation:** การเก็บข้อมูลลูกค้าสอดคล้องกับนโยบาย PDPA เบื้องต้น (ขอความยินยอมก่อนเก็บข้อมูล)

## ส่วนที่ลดทอนขอบเขต 

### 1. Mobile Application (Native) และ POS Terminal
* **รายละเอียดการเปลี่ยนแปลง:**
    * ยกเลิกการพัฒนา Native Mobile App (iOS/Android)
    * ยกเลิกการเชื่อมต่อกับเครื่องรูดบัตรเครดิต (POS Hardware Integration)
    * **คงเหลือ:** ระบบ Web Application ที่รองรับการใช้งานผ่าน Mobile Browser (Responsive Web)
* **เหตุผลประกอบ (Justification):**
    > **Feasibility & Focus:** การพัฒนาทั้ง Native App, Web App และเชื่อมต่อ Hardware ไปพร้อมกัน มีความซับซ้อนสูงและใช้เวลาเกินขอบเขตของรายวิชา การตัดส่วนนี้ออกทำให้ทีมพัฒนาสามารถโฟกัสที่ความสมบูรณ์ของ Core Logic บน Web Application ได้อย่างเต็มประสิทธิภาพและส่งมอบงานได้ตรงเวลา

### 2. ระบบ GPS Tracking แบบ Real-time
* **รายละเอียดการเปลี่ยนแปลง:**
    * ยกเลิกการติดตามตำแหน่งรถขนส่งแบบ Real-time บนแผนที่
    * **ปรับเปลี่ยนเป็น:** ระบบติดตามสถานะงาน (Status Tracking) และการยืนยันด้วยภาพถ่าย (Photo Proof) แทน
* **เหตุผลประกอบ (Justification):**
    > **Cost & Complexity:** การใช้งาน Google Maps API สำหรับ Real-time Tracking มีความซับซ้อนในการจัดการ Driver Side และอาจมีค่าใช้จ่าย API การเปลี่ยนมาใช้ระบบอัปเดตสถานะเป็นช่วงๆ (Milestone Tracking) เพียงพอต่อความต้องการบริหารจัดการงานซักรีดและประหยัดทรัพยากรระบบ

### 3. ระบบระดับสมาชิก (Tier System)
* **รายละเอียดการเปลี่ยนแปลง:**
    * ปรับลดระดับสมาชิกจากเดิม 3 ระดับ (Silver, Gold, Platinum)
    * **คงเหลือ:** สถานะสมาชิกทั่วไป และ สมาชิก VIP
* **เหตุผลประกอบ (Justification):**
    > **Logic Simplification:** เพื่อลดความซับซ้อนของ Algorithm ในการคำนวณส่วนลดและการเลื่อนระดับสมาชิก (Upgrade Logic) ทำให้ระบบมีความเสถียรและตรวจสอบความถูกต้องได้ง่ายขึ้น

### 4. ระบบ Barcode Scanner
* **รายละเอียดการเปลี่ยนแปลง:**
    * เปลี่ยนจากการใช้ระบบ Barcode แบบดั้งเดิม
    * **ปรับเปลี่ยนเป็น:** ระบบ QR Code
* **เหตุผลประกอบ (Justification):**
    > **Modernization & Cost:** QR Code มีความยืดหยุ่นสูงกว่า สามารถใช้กล้องสมาร์ทโฟนทั่วไปสแกนได้ทันทีโดยไม่ต้องลงทุนซื้ออุปกรณ์ Hardware (Barcode Gun) เพิ่มเติม และรองรับข้อมูลได้มากกว่า

###  ส่วนที่เพิ่มเติม

#### 1. ระบบตรวจสอบประวัติการใช้บริการ (Customer Service History)
* **สิ่งที่เพิ่ม:** ฟังก์ชันที่สามารถกดดูรายการซักรีดย้อนหลังของลูกค้าแต่ละคนได้
    * **แสดงข้อมูล:** วันที่ส่งผ้า, รายการเสื้อผ้าที่ส่ง, ยอดเงิน, และสถานะงานที่ผ่านมา
* **วัตถุประสงค์:** * เพื่อให้ทางร้านสามารถตรวจสอบข้อมูลย้อนหลังได้ทันทีเมื่อลูกค้าสอบถาม
    * เพื่อใช้อ้างอิงกรณีเกิดปัญหากับเสื้อผ้า หรือตรวจสอบรายการผ้าที่ส่งประจำ

#### 2. แดชบอร์ดสรุปยอดประจำวัน (Daily Summary Dashboard)
* **สิ่งที่เพิ่ม:** หน้าแรกที่แสดงตัวเลขสรุปง่ายๆ ให้ผู้ดูแลระบบเห็นทันทีที่ Login
    * จำนวนผ้าที่รับวันนี้ (ชิ้น)
    * จำนวนงานที่ค้างส่ง (งาน)
    * ยอดรายรับวันนี้ (บาท)
* **วัตถุประสงค์:** เพื่อให้เจ้าของร้านเห็นภาพรวมกิจการได้ทันที โดยไม่ต้องเสียเวลานับจากเอกสารกระดาษ

#### 3. ระบบค้นหาข้อมูล (Smart Search)
* **สิ่งที่เพิ่ม:** ช่องค้นหาที่สามารถพิมพ์ข้อมูลเพื่อค้นหาได้หลายรูปแบบในช่องเดียว
    * ค้นหาจาก: ชื่อลูกค้า, เบอร์โทรศัพท์, หรือ เลขที่ใบรับงาน
* **วัตถุประสงค์:** เพื่อความรวดเร็วในการให้บริการหน้าเคาน์เตอร์ พนักงานไม่ต้องเสียเวลาไล่หาชื่อลูกค้าจากรายชื่อทั้งหมด
