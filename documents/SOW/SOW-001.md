## 1. ที่มาและวัตถุประสงค์

### 1.1 ที่มาและปัญหา (Background & Problem Statement)
ในปัจจุบัน ธุรกิจร้านซักรีดขนาดย่อมและขนาดกลาง (SME) ส่วนใหญ่ยังคงใช้การบริหารจัดการด้วยมือ (Manual Process) ซึ่งก่อให้เกิดปัญหาเรื้อรังในการดำเนินงาน ได้แก่:
1.  **ความผิดพลาดของข้อมูล (Data Integrity):** การจดบันทึกด้วยกระดาษทำให้เกิดความคลาดเคลื่อนในการคำนวณราคา การนับจำนวนผ้า และการสรุปยอดรายรับ
2.  **ปัญหาการสูญหาย (Traceability):** ขาดระบบติดตามสถานะผ้าที่ชัดเจน ทำให้เกิดกรณีผ้าหาย หรือส่งคืนผิดเจ้าของ ซึ่งส่งผลเสียต่อชื่อเสียงร้าน
3.  **ความล่าช้าในการบริการ (Operational Inefficiency):** พนักงานใช้เวลาค้นหาข้อมูลลูกค้าและประวัติเก่านาน ทำให้เกิดคิวสะสมในช่วงเวลาเร่งด่วน
4.  **ขาดข้อมูลเพื่อการตัดสินใจ (Lack of Insight):** เจ้าของร้านไม่ทราบยอดขายที่แท้จริงแบบ Real-time ทำให้บริหารจัดการสต็อกและต้นทุนได้ยาก

### 1.2 วัตถุประสงค์ของงาน (Objectives)
1.  **เพื่อพัฒนาระบบร้านซักรีด** 
2.  **เพื่อเพิ่มทักษณะการทำงานร่วมกัน** 
3.  **เพื่อสร้างชิ้นงานที่สามารถนำไปใช้งานได้จริง**

## 2. ขอบเขตงาน (Scope of Work)

### 2.1 ในขอบเขต (In-Scope)
ระบบจะทำงานบนรูปแบบ Web Application โดยครอบคลุมฟังก์ชันหลักดังนี้:
1.  **ระบบจัดการลูกค้า (Customer Management):** รองรับการบันทึกข้อมูลส่วนตัว, ที่อยู่จัดส่ง, และระบบสมาชิก 
2.  **ระบบรับงานและจัดการออเดอร์ (Order Processing):** สร้างใบรับงาน, ระบุรายละเอียดผ้า คำนวณราคาอัตโนมัติ, และพิมพ์ใบรับงานพร้อม QR Code
3.  **ระบบติดตามสถานะ (Status Tracking):** อัปเดตสถานะงาน Realtime และลูกค้าสามารถสแกน QR ดูสถานะได้
4.  **ระบบค้นหาข้อมูล (Smart Search):** พนักงานสามารถค้นหาข้อมูลลูกค้าและรายการสั่งซื้อได้จาก ชื่อ, เบอร์โทรศัพท์
5.  **ระบบรายงานและผู้บริหาร (Dashboard):** แสดงกราฟสรุปยอดรับงานประจำวัน, งานค้างส่ง, รายได้รวมรายวัน รายเดือน 

### 2.2 นอกขอบเขต (Out-of-Scope)
เพื่อให้โครงการสามารถส่งมอบได้ตามระยะเวลาที่กำหนด จึงขอตัดส่วนงานดังต่อไปนี้ออก:
1.  การพัฒนา Mobile Application (Native App) บน iOS และ Android ใช้ Responsive Web แทน 
2.  การเชื่อมต่อกับอุปกรณ์ฮาร์ดแวร์เฉพาะทาง เช่น ลิ้นชักเก็บเงินอัตโนมัติ หรือเครื่องรูดบัตร 
3.  ระบบติดตามตำแหน่งรถขนส่งแบบ Real-time GPS 
4.  ระบบคำนวณเงินเดือนพนักงาน (Payroll) และระบบบัญชีภาษีเต็มรูปแบบ

### 2.3 สมมติฐาน (Assumptions)
1.  ผู้ใช้งาน (พนักงานและลูกค้า) มีอุปกรณ์ที่เชื่อมต่ออินเทอร์เน็ตได้ (สมาร์ทโฟน, แท็บเล็ต, หรือคอมพิวเตอร์)
2.  การใช้งาน QR Code จะใช้กล้องถ่ายรูปจากสมาร์ทโฟนของผู้ใช้งานแทนเครื่องสแกนบาร์โค้ด
3.  ระบบทำงานบนโครงสร้างพื้นฐาน Cloud Server ที่มีความเสถียร

### 2.4 ข้อจำกัด (Constraints)
* **ด้านเทคโนโลยี:** ระบบรองรับการทำงานบน Web Browser หลัก Chrome, Safari, Edge เวอร์ชันปัจจุบัน
* **ด้านระยะเวลา:** ต้องดำเนินการให้แล้วเสร็จภายใน 1 ภาคการศึกษา
* **ด้านกฎหมาย:** การเก็บรวบรวมข้อมูลส่วนบุคคลต้องมีระบบขอความยินยอม (Consent Form) ตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)

## 6. บทบาทและความรับผิดชอบ (Roles & Responsibilities)

### 6.1 ผู้เกี่ยวข้องหลักและทีมงาน
* **อาจารย์ที่ปรึกษา:** ผู้ให้คำปรึกษาทางวิชาการ, ผู้อนุมัติขอบเขต, และผู้ประเมินผลงาน
* **Project Manager (PM):** [นายอรัญชัย คำเพ็ญ]
    * รับผิดชอบการบริหารโครงการ, วางแผนงาน, ติดตามความคืบหน้า 
* **System Analyst & Backend Developer:** [นายวรปรัชญ์ บุญมี,นายอนาวิล บุญช่วย]
    * รับผิดชอบการวิเคราะห์ระบบ, ออกแบบฐานข้อมูล, และพัฒนาโปรแกรมส่วนหลังบ้าน API/Server
* **UX/UI Designer & Frontend Developer:** [นายสรวิชญ์ สมตน,นายวันขนะ ไชยลังกา] และ [นายอนาวิล บุญช่วย]
    * รับผิดชอบการออกแบบหน้าจอ (Design System), พัฒนาหน้าเว็บ Web Interface และเชื่อมต่อระบบ
* **System Analyst (SA) & Documentation:** [นายอรัญชัย คำเพ็ญ] และ [นางสาวชาญานิส ธีรเวโรจน์, นายณัฐพล จันทร์ฉาย]
    * รับผิดชอบการรวบรวมข้อมูล, จัดทำเอกสาร SRS, SOW

### 6.2 ตารางความรับผิดชอบ (RACI Matrix)

| กิจกรรม (Activity) | PM <br> (อรัญชัย) | Backend <br> (วรปรัชญ์) | Frontend/Design <br> (สรวิชญ์/วันขนะ) | BA/Docs <br> (ชาญานิส/ณัฐพล) | อาจารย์ |
|:---|:---:|:---:|:---:|:---:|:---:|
| **1. กำหนดขอบเขตโครงการ** | R/A | C | C | C | I/A |
| **2. จัดทำเอกสาร SRS / SOW** | A | I | I | R | I |
| **3. ออกแบบหน้าจอ (UI/UX)** | I | C | R | C | I |
| **4. พัฒนา Coding / Database** | I | R | R | I | I |
| **5. ทดสอบระบบ (Testing)** | A | C | C | R | I |
| **6. นำเสนอและส่งมอบงาน** | A | R | R | R | A |

*(R = ผู้ลงมือทำ, A = ผู้รับผิดชอบหลัก/อนุมัติ, C = ผู้ให้คำปรึกษา, I = ผู้รับทราบข้อมูล)*

## 7. การสื่อสารและการรายงาน (Communication Plan)

เพื่อให้การทำงานเป็นไปอย่างต่อเนื่องและมีประสิทธิภาพ ทีมงานได้กำหนดข้อตกลงในการสื่อสารดังนี้:

* **ช่องทางสื่อสารหลัก (Internal):** Discord Server สำหรับการคุยงานและแชร์หน้าจอ
* **การจัดการซอร์สโค้ด (Version Control):** ใช้ **GitHub** ในการเก็บไฟล์งานและรวมโค้ด Merge Branch
* **การประชุมทีม (Daily Stand-up):** ประชุมออนไลน์ 5 วันต่อสัปดาห์ เวลา 20:00 น. เป็นต้นไป 
* **การรายงานความคืบหน้า (Advisor Meeting):** นัดพบอาจารย์ที่ปรึกษา ทุกวันพฤหัสบดี ตามเวลาเรียน
