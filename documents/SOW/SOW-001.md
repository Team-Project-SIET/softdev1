## 1. ที่มาและวัตถุประสงค์

### 1.1 ที่มาและปัญหา (Background & Problem Statement)
ปัจจุบันธุรกิจร้านซักรีดขนาดย่อมมักประสบปัญหาจากการบริหารจัดการด้วยมือ (Manual Process) 
* **ความผิดพลาดของข้อมูล:** การจดบันทึกใส่สมุดทำให้เกิดความผิดพลาดในการคำนวณราคาและสรุปยอด
* **การสูญหายของพัสดุ:** ไม่มีระบบติดตามสถานะผ้าที่ชัดเจน ทำให้เกิดกรณีผ้าหายหรือสลับเจ้าของ
* **การบริการล่าช้า:** การค้นหาข้อมูลลูกค้าและประวัติเก่าทำได้ยากและใช้เวลานาน
* **กลุ่มผู้ได้รับผลกระทบ:** เจ้าของร้าน, พนักงานต้อนรับ, และลูกค้าที่ใช้บริการ

### 1.2 วัตถุประสงค์ของงาน (Objectives)
1.  เพื่อพัฒนาระบบ Web Application สำหรับบริหารจัดการงานรับ-ส่งผ้าและคำนวณค่าบริการ
2.  เพื่อลดข้อผิดพลาดในการปฏิบัติงานและป้องกันการสูญหายของเสื้อผ้าด้วยระบบ QR Code
3.  เพื่ออำนวยความสะดวกให้ลูกค้าสามารถตรวจสอบสถานะและประวัติการใช้บริการได้ด้วยตนเอง
4.  เพื่อสรุปรายงานรายรับ-รายจ่าย และข้อมูลการดำเนินงานให้ผู้บริหารทราบได้ทันที

## 2. ขอบเขตงาน (Scope of Work)

### 2.1 ในขอบเขต (In-Scope)
ระบบจะครอบคลุมฟังก์ชันการทำงานหลัก ดังนี้:
1.  **ระบบจัดการลูกค้า:** สมัครสมาชิก, บันทึกข้อมูลที่อยู่, และระบบสมาชิก VIP
2.  **ระบบรับงาน (Order):** สร้างใบรับงาน, คำนวณราคาอัตโนมัติ, และพิมพ์ใบรับงานพร้อม QR Code
3.  **ระบบติดตามสถานะ:** อัปเดตสถานะงาน (รับผ้า -> กำลังซัก -> เสร็จสิ้น) และตรวจสอบประวัติย้อนหลัง
4.  **ระบบค้นหาข้อมูล:** ค้นหาลูกค้าและออเดอร์จาก ชื่อ, เบอร์โทร, หรือรหัสใบงาน
5.  **ระบบรายงาน (Dashboard):** เจ้าของร้าน แสดงยอดรับงานประจำวัน, งานค้างส่ง, และรายได้รวม ลูกค้า 
---
### 2.2 นอกขอบเขต (Out-of-Scope)
เพื่อให้โครงการเสร็จทันตามกำหนดการ จึงขอตัดส่วนงานที่ไม่จำเป็นเร่งด่วนออก ดังนี้:
1.  การพัฒนา Mobile Application (Native App) บน iOS และ Android
2.  การเชื่อมต่อกับอุปกรณ์ฮาร์ดแวร์ POS หรือลิ้นชักเก็บเงินอัตโนมัติ
3.  ระบบติดตามรถขนส่งแบบ Real-time GPS (ใช้ระบบอัปเดตสถานะแทน)
4.  ระบบคำนวณเงินเดือนพนักงาน (Payroll) และระบบบัญชีภาษีเต็มรูปแบบ
